# Progress Tracking Rules (Unstable Studio)

## Workflow
1. Create planning first: `planning/PLANNING.md`
2. Work in small items (status: 📋 → 🔄 → ✅)
3. For every file change: log it in `logs/CHANGELOG.md`
4. When done: check all items + write a summary + run self-analysis

## Changelog Rules
- Record files created/edited/removed
- Record lines: + (add), ~ (edit), - (remove)
- Record the reason for changes

## Definition of Done
- All todos completed
- No crashes in primary flows
- No editor lag for thousands-of-lines scenarios (baseline)
- LSP basic flows work (initialize, didOpen, didChange, completion, diagnostics)
